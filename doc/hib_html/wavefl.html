Flag WAVEFL

If WAVEFL is set TRUE, then the adiabatic energies and transformation matrices are computed for the subsequent calculations. These are written (in binary) to the file {jobname}.wfu.

Subsequent calculation of the wavefunction is invoked by the command PSI. Subsequent calculation of the incoming and outgoing fluxes is controlled by the command FLUX. In addition, the command EADIAB can be invoked to print out adiabatic energies as a function of R.

The WRSMAT flag controls the level of output in the wfu file. If WRSMAT is set FALSE, only adiabatic energies will be stored and EADIAB is the only subsequent calculation available. If If WRSMAT is set TRUE, all three subsequent calculations mentioned above can be invoked.

The total storage required to store the transformation matrix when WRSMAT is set TRUE is

(Nlog + 2 Nairy) x Nch2 + 3 Nairy x Nch,

where Nlog and Nairy are the number of LODG and AIRY integration steps and Nch is the number of channels. Thus the file {jobname}.wfu can often be very large.
